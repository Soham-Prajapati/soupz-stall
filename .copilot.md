# Copilot Project Memory Guide

## Purpose
This file is the persistent handoff memory for all future agents working in this repository.
Use it as the first source of truth before making decisions, edits, or architecture changes.

## Project Intent
- Product: Soupz Stall multi-agent orchestration system.
- Primary UX direction: move from terminal-heavy interaction to web-first control panels.
- Design metaphor: open food-stall yard operations (3-4 stalls, lanes, chefs, order flow, service pass).
- Runtime direction: local-first execution with optional remote access via OTP pairing.

## Core Architecture (Current)
- Orchestrator: src/orchestrator/router.js and src/orchestrator/semantic-router.js.
- Agent execution: src/agents/spawner.js.
- Interactive CLI session logic: src/session.js.
- Remote bridge (REST + WebSocket + OTP): packages/remote-server/src/index.js.
- Dashboard UI (React/Vite): packages/dashboard/src/App.jsx.
- Session monitor and telemetry state: src/core/stall-monitor.js.

## Working Preferences For Future Agents
- Prefer web UX improvements over terminal-only UX additions when both are possible.
- Preserve open food-stall yard metaphor in naming and UI language.
- Treat kitchen internals as per-stall workspaces, not the global scene.
- Keep local execution as default. Remote mode must remain opt-in and authenticated.
- Do not remove CLI capabilities; treat CLI as backend/operator fallback.

## Memory Rules For Future Agents
1. Read this file first.

## Model Strategy Baseline
- Assume premium/pro models may be unavailable or rate-limited.
- Implement and preserve fallback chains:
  - primary preferred model,
  - lower-cost cloud fallback,
  - local model fallback (Ollama) where acceptable.
- Surface fallback reason in UI/reporting whenever model changes at runtime.

## UX Baseline For Web Transition
- Required web primitives:
  - prompt composer text area,
  - execution timeline,
  - streamed output panel,
  - queue/history,
  - charts (latency, success rate, token/cost),
  - model and fallback indicators.
- UI should feel like a kitchen control room, not a generic chatbot clone.

## Visual Baseline Update
- Production asset mode: strict top-down stall pack.
- Isometric assets are experimental and isolated.
- Chroma background (#00FF00) is default export to avoid fake transparency artifacts.

## Security Baseline
- Keep OTP pairing mandatory for remote control.
- Keep authenticated routes separated from public health/pairing discovery endpoints.
- Add execution limits in web mode (timeout, output size caps) to reduce abuse.
