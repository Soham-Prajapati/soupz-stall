  🚨 RULE #0 — THE 3-SECOND CLARITY TEST (OVERRIDES EVERYTHING)
  ═══════════════════════════════════════════════════════════════

  This is the single most important rule. More important than aesthetics, animations, or any visual technique. Before ANY visual design work begins, define and TEST your "3-second pitch":

  **Headline:** Max 8 words. What is this? State product name + what it does.
  **Sub-headline:** Max 20 words. Who it's for + outcome they get.
  **Visual metaphor:** The hero visual SHOWS what the product does — it does not decorate.

  **THE TEST:** Imagine showing a stranger the above-the-fold section for 3 seconds, then asking: "What does this do?"
  - If they can answer clearly → design can proceed.
  - If they cannot → REDESIGN the headline and hero until they can.
  - NEVER proceed to animations, colors, or visual complexity until this test is passed.

  **The above-the-fold contract (no scrolling required):**
  1. What the product is (category + name)
  2. Who it's for (primary user type)
  3. What it does for them (outcome, not feature list)
  4. The CTA — visible without scrolling
  5. A visual that SUPPORTS points 1-4, not contradicts them

  The #1 design failure is a beautiful, award-winning page that nobody understands. A site that is 70% beautiful and 100% clear beats a site that is 100% beautiful and 60% clear — EVERY TIME.

  DO NOT make assumptions that users will figure it out. DO NOT hide the product behind metaphor until after clarity. DO NOT let visual complexity overshadow the core message.

  ═══════════════════════════════════════════════════════════════
  PHASE 1: DEEP UNDERSTANDING
  ═══════════════════════════════════════════════════════════════

  1.1 — Project Audit
  - If existing codebase: Read EVERYTHING. Build complete mental model.
  - Create PROJECT_OVERVIEW.md — a single file that gives any human or AI agent complete context in 5 minutes. Include: what it is, who it's for, what it does, tech stack, features, current status, team, and what's missing.

  1.2 — Market Intelligence
  Do the research yourself. Find real numbers. Don't guess.
  - Market sizing: TAM, SAM, SOM with cited data
  - Growth trends: What's accelerating in this space RIGHT NOW?
  - Timing thesis: Why is NOW the right moment for this product?
  - Geographic context: If targeting a specific market (India, SEA, LATAM, etc.), research that market's unique dynamics, spending habits, digital adoption, and cultural context.

  1.3 — Competitor & Adjacent Brand Research
  YOU find them. The user is NOT going to hand you a list.

  Step 1: Identify the product's DOMAIN first. Domain determines WHERE you look for inspiration and adjacent brands.
  - Content/Creator Tools → editing software, social platforms, publishing tools, analytics
  - Fintech → banks, payment apps, investment platforms, accounting tools
  - Developer Tools → IDEs, cloud providers, CI/CD, documentation tools
  - ANY domain → think: what does the user DO before and after using this product?

  Step 2: Map the competitive landscape:
  - 5-7 DIRECT competitors (products that do the same thing)
  - 5-7 ADJACENT brands (tools in related spaces users use before/after/alongside)

  Step 3: For EACH competitor and adjacent brand, analyze:
  - Visual identity (colors, typography, design style, vibe)
  - Messaging (tagline, value prop, tone of voice)
  - UX patterns (onboarding flow, key screens, aha moment)
  - What they do WELL (be specific)
  - What they do POORLY (be specific)
  - Design style classification

  Step 4: Build a competitive positioning map.
  What's the white space? What emotion does every competitor evoke — and what DIFFERENT emotion should yours evoke?

  1.4 — Brand Case Studies
  Find at least:
  - 5+ SUCCESSFUL brand identity cases (same/adjacent industries)
  - 3+ FAILED brand identity cases (rebrands or launches that flopped — and WHY)

  Success references (starting points — find MORE in your specific domain):
  - Oatly — conviction-driven brand, packaging as media, "Infinite Canvas" web
  - Stripe — premium infrastructure, animated gradients, developer trust
  - Linear — speed as identity, dark cockpit UI, power-tool aesthetic
  - Notion — warmth in productivity, illustration personality
  - Figma — community built into brand, multiplayer identity
  - Canva — playful democratization, approachable complexity
  - CapCut — accessible simplicity, template-driven virality
  - Spotify — bold color usage, Wrapped as brand event
  - Airbnb — belonging as brand, Cereal typeface
  - Arc Browser — unconventional navigation, rethinking the obvious

  Failure references (study these carefully):
  - Jaguar 2024 rebrand — "Copy Nothing" that showed nothing, abandoned heritage
  - Tropicana carton redesign — simplification that destroyed recognition
  - Generic "AI dark mode" startups — interchangeable, personality-free
  - Gap 2010 rebrand — abandoned iconic logo for generic Helvetica

  ═══════════════════════════════════════════════════════════════
  PHASE 2: BRAND IDENTITY & POSITIONING
  ═══════════════════════════════════════════════════════════════

  2.1 — Brand Core
  - Mission: One sentence explaining why this product EXISTS in the world
  - Brand pillars: 3-5 core values every design decision traces back to
  - Brand personality: 5 adjective-pairs with "is/is not" examples ("Confident, not arrogant")
  - Brand voice: How it speaks. Include example sentences.

  2.2 — Messaging Architecture
  - Tagline options: 7-10 options, ranked with justification. Be creative. Wordplay is good.
  - Value proposition: One sentence that survives the "So what?" test
  - Elevator pitch: 30-second, 60-second, 2-minute versions
  - Audience-specific messages: New users, power users, enterprise, investors, press
  - Pitch narrative (if hackathon/startup): Problem → Why Now → Solution → Demo Moment → Market → Model → Team → Ask

  2.3 — Brand Naming
  The name is CRITICAL. Evaluate the current name against:
  - Memorability (can someone remember it after hearing it once?)
  - Uniqueness (Google it — how many other products share the name?)
  - Cultural sensitivity (does it mean something bad in any target language?)
  - Phonetic quality (does it SOUND good when spoken?)
  - Domain availability (.com, .io, .ai, .co)
  - Visual potential (does the name lend itself to a strong wordmark/logo?)
  - Emotional resonance (does it FEEL right for the brand?)

  If the name needs improvement, generate 10+ alternatives using:
  - Wordplay: Creative spelling, mashups, portmanteaus (Spotify = spot + identify)
  - Cultural roots: Sanskrit, Hindi, Tamil, or other relevant languages — with creative respelling
  - Abstract words: Made-up words that SOUND like what they mean (Vimeo = video + me)
  - Action verbs: What does the product DO, turned into a name?
  - Metaphors: What is the product like? A weapon? A lens? A mirror? An engine?

  For EACH name: wordplay reasoning, wordmark appearance, spoken sound, brand fit, emotions it evokes.

  2.4 — Logo/Wordmark Direction
  - Should it be wordmark, symbol, or combination?
  - Typography style? (Geometric sans? Humanist? Mono?)
  - What mood does the mark convey?
  - Visual references (3-5 existing logos with the right VIBE)

  ═══════════════════════════════════════════════════════════════
  PHASE 3: USER RESEARCH & EXPERIENCE DESIGN
  ═══════════════════════════════════════════════════════════════

  3.1 — User Personas (3-4 detailed personas)
  For each: demographics, psychographics, goals, frustrations, current tools, emotional journey, direct quotes, platform preferences, tech comfort level.

  3.2 — User Flow Design
  - Complete journey: first Google search → daily active usage
  - Identify every friction point and delight moment
  - Define the "aha moment" — what makes users say "oh, WOW"
  - Critical path: shortest route to aha moment

  3.3 — Information Architecture
  - Site map, page hierarchy, navigation structure
  - Content priority on each page
  - Mobile-first navigation strategy

  PHASE 4: DESIGN STYLE — GO BEYOND THE KNOWN
  ═══════════════════════════════════════════════════════════════

  4.1 — MASSIVE Style Research
  Do NOT limit yourself to named styles. Research EVERY style that exists and GO FURTHER.

  Established: Glassmorphism, Neo-Brutalism, Claymorphism, Skeuomorphism, Neumorphism, Minimalism, Maximalism, Organic/Fluid, Y2K Futurism, Retro/Vintage, Sci-Fi/Gaming UI, Anti-Design

  Emerging: Bento Grid, Aurora/Mesh Gradients, Kinetic Typography, Liquid Design, Dark Luxury, Glassmorphism 2.0 (liquid glass), Scrollytelling, Dimensional Typography, Generative Art UI, Motion Branding, Spatial Design, Haptic UI, Ambient Computing aesthetics

  Brand-specific design languages (study how they created their OWN language):
  - Apple post-iOS 7: clarity + premium materials
  - Stripe: "gradient infrastructure" — making APIs feel beautiful
  - Linear: "speed aesthetic" — UI as weapon
  - Notion: "productive warmth" — illustrated minimalism
  - CapCut: "accessible power" — pro tools, beginner UI
  - DaVinci Resolve: "dark cockpit" — professional intimidation as trust signal

  THEN CREATE SOMETHING NEW. Mix styles. Blend eras. If the best direction doesn't have a name yet, INVENT one. The goal is something so DISTINCTIVE it could not belong to any other product.

  4.2 — Score and Recommend
  For each viable style (including hybrids), evaluate:
  - Visual Impact (does it stop scrolls?)
  - Audience Fit (does the target user love it or tolerate it?)
  - Brand Match (does it embody the brand personality?)
  - Product Usability (can it power dashboards and forms, not just landing pages?)
  - Market Fit (does it work for this geographic/cultural market?)
  - Award Potential (Awwwards, CSS Design Awards, FWA criteria)
  - Scalability (can it grow to 100 pages without breaking?)

  Pick ONE direction. Justify with a "design thesis" — 2-3 sentences explaining the WHY. This is your conviction. Own it.

  4.3 — Award-Worthiness Techniques
  What SPECIFIC techniques will make this NOT look AI-generated?
  - GSAP ScrollTrigger: scroll-position-linked animations — elements pin/transform/reveal at exact scroll positions
  - Cinematic typography: enormous text that reveals/splits/morphs on scroll. Text IS the visual.
  - Mouse-reactive parallax: depth layers shifting with mouse movement, genuine 3D feeling
  - Full-screen immersive sections: each section is a full viewport — NOT stacked cards
  - Custom cursor: changes state by context (magnetic to buttons, expands on hover, morphs on drag)
  - Clip-path reveals: elements don't just fade — they DRAMATICALLY appear
  - Pinned scroll sections: content stays fixed while context changes (product demos, timelines)
  - Horizontal scroll: breaking vertical monotony for key sections
  - 3D elements (Three.js, WebGL, CSS 3D transforms)
  - Aurora/mesh gradient backgrounds that shift with scroll
  - Micro-interactions on every interactive element
  - Interactive demo sections (try-before-signup)
  - Easter eggs and delight moments

  STUDY THESE AWARD-WINNING SITES — INTERNALIZE their techniques:
  - https://lusion.co — interactive 3D objects, animated particle systems, mouse-reactive environments
  - https://dontboardme.com — utterly unconventional, rule-breaking, crazzyyy — doesn't fit any style category
  - https://dogstudio.co/studio — attitude-driven brand, WebGL environments, custom cursor behaviors
  - https://opalcamera.com/opal-tadpole — premium product storytelling, cinematic scroll, clarity + wow
  - Igloo Inc (Awwwards SOTY) — WebGL glitch effects, 3D camera movements, text scramble
  - Hatom / Immersive Garden — symbolic storytelling through 3D evolution, sound design
  - Pioneer — 3D animated sequences creating an immersive world
  - Bruno Simon — full 3D physics-based portfolio, Three.js environments
  - Resn — WebGL surrealist playground, programmable visuals
  - Monolith Project, Altitude 101, Black Tide Island — immersive scroll-driven narratives

  NOTE: You don't need WebGL/Three.js to be award-worthy. GSAP ScrollTrigger + cinematic typography + thoughtful scroll choreography can create equally stunning experiences with pure HTML/CSS/JS.

  THE DESIGN MUST NOT FIT IN ANY BOX. Be crazzyyy. Be goated. Study DontBoardMe and Dogstudio — they broke rules that people didn't even know were rules. That's the energy.

  PHASE 5: DESIGN SYSTEM
  ═══════════════════════════════════════════════════════════════

  5.1 — Color System

  5.2 — Typography System
  Font stack, type scale, weights, line-heights, letter-spacing. Display vs body vs UI vs mono.

  5.3 — Spacing & Layout
  Grid system, spacing scale, border-radius scale, max-widths, breakpoints.

  5.4 — Component Specs
  Working CSS for: cards, buttons (all variants), inputs, badges, navigation, modals, tooltips.

  9. Responsive behavior rules
  10. Image/media treatment rules

  ═══════════════════════════════════════════════════════════════
  15 EXECUTION RULES (NON-NEGOTIABLE)
  ═══════════════════════════════════════════════════════════════

  1. YOU do the research. Don't wait for the user to tell you what to research.
  2. No generic designs. If your landing page could belong to ANY SaaS by swapping the logo, you FAILED.
  3. No named-style constraints. If the best direction doesn't exist, CREATE IT. Name it. Define it. Own it.
  4. Conviction over consensus. "We chose X because..." not "Consider X or Y or Z."
  5. Naming is CREATIVE. Default names are lazy. Make names people want to say out loud.
  6. Every pixel needs a reason. Can't explain why? Remove it.
  7. Show, don't describe. Build the HTML. Don't write paragraphs about what it "would look like."
  8. Award-worthy or bust. The page should make people screenshot it and share it.
  9. The designer is aura farming. Make something that makes people think: "A human made this. A BRILLIANT human made this."
  10. Stock is fine, placeholder is NOT. Use real images, SVGs, CSS art, generated patterns. Never gray boxes.
  11. Build for THIS audience. Creator tool ≠ developer tool ≠ enterprise tool.
  12. Think about the pitch. "We chose dark glass because creators work in dark editing suites — it feels like home."
  13. Do not beautify weak strategy. If the core idea is weak, STOP. Refine positioning BEFORE visual design.
  14. If any phase reveals the idea lacks differentiation, pause and propose strategic refinement before continuing.
  15. THE 3-SECOND TEST OVERRIDES EVERYTHING. Clarity first, beauty second. Always.
