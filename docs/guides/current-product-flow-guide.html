Soupz Current Product Flow Guide

Complete runtime map of the current product. Built for human readability first.

Date: 2026-03-15 Mode: Local-first + optional remote Design metaphor: Kitchen control room

1) What You Run Now (Single Command)

Recommended startup:

pnpm run dev:web:pnpm

Fallback:

npm run dev:web

What happens automatically

  • Remote server starts
  • Pairing code is generated
  • Pairing code is validated into token
  • Dashboard starts
  • Browser opens with token + remote URL prefilled

What you should not do manually anymore

  • No manual 4-terminal setup
  • No manual OTP copy dance each run
  • No manual token paste unless you want custom setup

2) End-to-End Product Flow (Current)

User prompt in dashboard → POST /api/orders on remote server → server launches CLI ask path (single-agent safe mode by default for web auto) → stdout/stderr captured as order output/events → dashboard polls orders + order detail → timeline/queue/output update in UI.

Frontend (Dashboard)

  • Prompt composer
  • Queue list
  • Timeline events
  • Output panel
  • File diff drawer

Remote Server

  • OTP + token auth
  • Order APIs
  • Change/diff APIs
  • WebSocket terminal bridge

Core Runtime

  • CLI orchestration
  • Agent spawner
  • Router and model policy
  • Memory and grading

3) Agent Types You Asked About

soupz-orchestrator

Task-level conductor. Best when you want one request decomposed into clear specialist subtasks.

  • Focused delegation logic
  • Good for planning + synthesis
  • Lower ceremony

soupz-workflow

Workflow-mode wrapper. Best when you want structured pipelines such as party mode or quick-dev sequences.

  • Broader orchestration framework
  • Higher automation breadth
  • Useful for larger multi-step workflows

6) Kitchen UI Gemini Prompt (Room Layout Based On Your Mockup)

Create a responsive kitchen control room dashboard UI using a top-down room metaphor.

Reference composition:
- Window centered at top wall.
- Door on left side.
- Two top countertops (left/right).
- Center operator zone with an "agent specific prompt" panel.
- Long bottom "general prompt" command bar.
- Right side timeline stack.
- Left/middle queue and station cards.

Behavioral UI requirements:
- General prompt sends one safe default agent by default.
- Agent-specific prompt targets selected specialist.
- Show lifecycle events: order.created, route.selected, chef.started, chef.output.delta, order.completed/failed.
- Include file-change drawer with diff preview.

Visual direction:
- Kitchen operations style, high contrast, clean outlines, no visual clutter.
- Palette: steel #5B6770, cream #F3E9D2, charcoal #222222, orange #E86A33, green #4CAF50, red #D64545, teal #2FA7A0.
- Bold, readable typography and intentional spacing.
- Mobile responsive under 768px with stacked layout.
