## 🏗️ System Architecture

`soupz-agents` is a local-first multi-agent orchestration platform designed for observability and efficiency.

### Core Components
- **CLI Runtime (`bin/soupz.js`, `src/`)**: The engine that handles routing, session management, and agent execution.
- **Remote Server (`packages/remote-server`)**: A Node.js/Express bridge that exposes CLI functionality over HTTP/WebSockets with PTY support.
- **Web Dashboard (`packages/dashboard`)**: A React-based UI for monitoring and interacting with agents.
- **Agent Registry**: A collection of specialist personas (`defaults/agents/`) layered on top of headless agents (Copilot, Gemini).
- **Orchestrator**: Handles semantic routing and multi-agent coordination (decomposition, parallel dispatch, synthesis).

## 🧩 Key Concepts

### 1. Hooks (Lifecycle Extension)
Hooks are extension points where logic is injected into the agent's workflow:
- **Pre-execution**: `ContextPantry.recall()` (memory injection), `OllamaPreprocessor` (prompt optimization).
- **Execution**: `AgentSpawner.on('output')` (real-time stream processing).
- **Post-execution**: `Grading.recordResult()` (quality assessment), `MemoryPool.store()` (persistence), `processDelegations()` (task chaining).

### 2. MCP (Model Context Protocol)
Located in `src/mcp/client.js`. It allows `soupz-agents` to connect to external MCP servers (like `npx @stitch/mcp-server`), enabling agents to use tools provided by those servers.
- **Register**: `/mcp register <name> <cmd>`
- **Connect**: `/mcp connect <name>`
- **Call**: `/mcp call <server> <tool> <args>`

### 3. Memory & Context
- **ContextPantry (`src/core/context-pantry.js`)**: Manages the immediate conversation context.
- **MemoryPool (`src/memory/`)**: Long-term storage for task results, enabling cross-session continuity.
- **TokenCompressor**: Shortens long prompts to save tokens while preserving meaning.

### 4. Plan Mode (Task Decomposition)
Implemented via the `todoList` in `src/session.js`.
- If a prompt is detected as "complex" (multi-step), it is automatically broken into a task list.
- Users can manage tasks with `/todo`, `/do 1`, and `/do all`.
- Orchestration uses `orchestrator.decompose()` to split a prompt into sub-tasks for parallel execution.

### 5. Skills
Defined in `src/skills.js`. Skills are discoverable specialist agents (personas) that can be invoked via `@name`. They are essentially pre-configured system prompts that wrap headless agents.

## 🚀 Efficiency Strategies (Inspired by Ruflo)

- **Smart Routing**: Route simple tasks to `qwen2.5:1.5b` (Ollama) or regex-based rules. Reserve premium models for complex reasoning.
- **Token Optimization**: Use `TokenCompressor` and only include relevant context from the `ContextPantry`.
- **Parallelism**: Use `orchestrateMultiAgent` or `/fleet` to run sub-tasks in parallel across different engines (Copilot and Gemini) to reduce total execution time.

## 🎨 UI/UX & Design Standards

### 1. Design Rules (Rule #0: Beauty never trumps understanding)
- **Limited Color Palette**: Accent + semantic colors only. No decorative colors.
- **Fewer Font Weights**: Use 400, 500, 600 only.
- **Generous Spacing**: 8px (items), 16px (groups), 24px (sections).
- **Borders over Shadows**: Use borders for structure; shadows only for overlays.
- **No Decorative Shapes**: Every element must serve a purpose.
- **Consistent Radius**: sm(4px), md(6px), lg(8px), xl(12px).

### 2. Theming & CSS
- **CSS Variables**: Use `--*-ch` space-separated RGB channel vars (e.g., `--accent-ch: 99 102 241`) for Tailwind opacity compatibility (`bg-accent/10`).
- **Themes**: 12 supported themes (dark, dim, midnight, light, monokai, nord, dracula, catppuccin, tokyo-night, rose-pine, solarized, github-dark).
- **Fonts**: `font-ui` for interface, `font-mono` for code.

### 3. Coding Conventions
- **No Emojis**: Avoid emojis in code or agent responses unless explicitly requested.
- **Concise & Direct**: No fluff.
- **Icons**: Use **Lucide** icons consistently.
- **Persistence**: Store all UI state in `localStorage`.
- **Naming**: Agent IDs must be `kebab-case` (e.g., `claude-code`, `ai-engineer`).
- **Git**: Commits should include `Co-Authored-By: Soupz <agent@soupz.vercel.app>`.
